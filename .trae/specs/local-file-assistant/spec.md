# 本地电脑助手产品规格文档

## 1. 产品概述

### 1.1 产品名称
本地电脑助手 (Local File Assistant)

### 1.2 产品定位
一款面向Windows/Mac平台的桌面应用程序，提供文件智能整理、局域网文件传输和C盘空间优化三大核心功能。

### 1.3 目标用户
- 普通家庭用户：管理个人照片、视频、文档
- 办公人员：电脑与手机间传输工作文件
- 摄影爱好者：批量整理和浏览照片
- 电脑初学者：安全清理C盘空间

## 2. 问题陈述

### 2.1 当前痛点
1. **文件管理混乱**：用户电脑中存储的图片和文档日益增多，手动整理耗时费力
2. **跨设备传输不便**：现有方案（微信、QQ）依赖外网，速度受限
3. **C盘空间告急**：缺乏专业指导清理磁盘，容易误删系统文件

### 2.2 解决方案
- 智能文件分类和重复文件检测
- 局域网内快速文件传输（无需互联网）
- 智能C盘分析和安全清理建议

## 3. 功能需求

### 3.1 文件整理模块

#### 3.1.1 智能分类
- **图片分类**：根据EXIF信息或修改时间，自动归类到"年/月/日"结构
- **文档分类**：按文件类型（PDF、Word、Excel）分类
- **用户自定义规则**：支持按文件名关键字、文件大小等创建规则

#### 3.1.2 重复文件检测
- 基于MD5哈希值检测重复文件
- 列出重复文件组，支持选择性删除或移动

#### 3.1.3 时间线浏览
- 以时间轴（日历）形式展示文件
- 支持按年、月、日缩放浏览

#### 3.1.4 批量重命名
- 支持按规则批量重命名（如"日期_序号"格式）

### 3.2 局域网文件传输模块

#### 3.2.1 传输模式
- **电脑→手机**：选择文件发送给已连接手机
- **手机→电脑**：手机浏览电脑共享文件夹并下载

#### 3.2.2 设备发现与连接
- 自动扫描局域网内设备
- 支持二维码快速连接

#### 3.2.3 传输管理
- 多文件并发传输
- 显示传输进度、速度、剩余时间
- 传输记录保存

#### 3.2.4 安全性
- 传输过程采用AES加密
- 临时验证码/密码授权机制

### 3.3 C盘优化模块

#### 3.3.1 磁盘空间分析
- **可视化分析**：饼图/树状图展示空间占用
- **大文件查找**：列出占用空间最大的前100个文件
- **按类别统计**：临时文件、日志、缓存等分类统计

#### 3.3.2 安全清理建议
- **风险等级标识**：
  - 🟢 绿色（安全可删）：临时文件、回收站、浏览器缓存
  - 🟡 黄色（谨慎删除）：应用缓存、日志文件、休眠文件
  - 🔴 红色（不建议删）：系统关键文件夹
- **影响说明**：详细说明删除后的影响

#### 3.3.3 一键清理（安全模式）
- 自动选择所有绿色项目清理
- 清理前二次确认
- 支持从回收站还原

#### 3.3.4 系统优化建议
- 检测并建议关闭休眠功能
- 检测虚拟内存设置
- 管理系统还原点

## 4. 非功能需求

### 4.1 性能要求
- 启动时间 ≤ 5秒
- 扫描10000个文件 ≤ 30秒
- 局域网传输速度 ≥ 理论带宽80%
- 空闲内存占用 < 100MB
- C盘扫描分析 ≤ 2分钟

### 4.2 易用性要求
- 界面简洁，符合主流操作习惯
- 向导式初次使用指引
- 关键操作有确认提示
- 风险标识一目了然

### 4.3 可靠性要求
- 文件整理支持回滚
- 传输断网自动重试
- C盘清理前创建系统还原点
- 不导致文件损坏或丢失

### 4.4 兼容性要求
- **电脑端**：Windows 10/11, macOS Monterey+
- **手机端**：Android 9+, iOS 13+（浏览器访问）
- **文件系统**：NTFS, APFS, FAT32
- **C盘优化**：主要针对Windows

### 4.5 安全性要求
- 不收集用户隐私数据
- 局域网传输需授权确认
- C盘清理需多次确认
- 高风险操作强制确认

## 6. 技术架构

### 6.1 技术选型
- **框架**：Electron（跨平台，Web技术）
- **前端**：HTML/CSS/JavaScript + 现代UI框架
- **后端**：Node.js
- **通信**：HTTP/REST + WebSocket
- **数据库**：SQLite

### 6.2 系统架构
```
+---------------------+
|   Electron App      |
|  +---------------+  |
|  |  Renderer     |  |
|  |  (UI Layer)   |  |
|  +---------------+  |
|  +---------------+  |
|  |  Main Process |  |
|  |  (Business)   |  |
|  +---------------+  |
+---------------------+
         |
    +----+----+--------+
    |         |        |
SQLite   File System  Network
```

### 6.3 核心模块
1. **FileOrganizer**：文件整理服务
2. **ServerManager**：局域网传输服务
3. **DiskOptimizer**：C盘优化服务
4. **DatabaseManager**：数据存储管理
5. **ConfigManager**：配置管理

## 7. 项目计划

### 7.1 里程碑
- **M1**：基础架构搭建 + 文件整理核心功能
- **M2**：局域网传输功能
- **M3**：C盘优化功能
- **M4**：UI优化 + 测试 + 发布

### 7.2 成功指标
- 文件整理准确率 ≥ 95%
- 局域网传输成功率 ≥ 99%
- C盘清理安全率 100%（不误删系统文件）
- 用户满意度 ≥ 4.5/5

## 8. 风险与缓解

### 8.1 技术风险
- **风险**：C盘清理误删系统文件
- **缓解**：严格的风险等级标识 + 多次确认 + 系统还原点

### 8.2 业务风险
- **风险**：用户不信任自动清理
- **缓解**：透明的影响说明 + 可还原机制
